# README Senior-Friendly Guide Design

## Goal

`README.md`를 Windows 11 초보 사용자, 특히 컴퓨터 사용이 익숙하지 않은 고령 사용자도 따라 할 수 있도록 다시 쓴다. 맨 위에는 "설치와 실행만 빠르게 따라 하는 안내"를 두고, 아래에는 기존 사용자나 개발자를 위한 고급 설명을 남긴다.

## User Experience

가장 위에는 어려운 개발 용어를 줄인 "아주 쉬운 시작 방법"을 둔다. 사용자는 `run_novel_autowriter.bat`를 더블클릭하는 방법을 가장 먼저 보게 되고, 그 다음으로 처음 한 번만 필요한 설치 절차를 읽게 된다. 명령어가 꼭 필요한 경우에도 "검은 창", "복사해서 붙여넣기", "엔터 누르기"처럼 익숙한 표현으로 설명한다.

중간에는 "처음 실행 후 할 일"과 "문제가 생기면"을 넣는다. 여기서는 API 키 방식과 Gemini CLI 방식 중 실제 앱이 지원하는 선택지를 간단하게 안내하고, 어떤 버튼을 누르면 되는지 설명한다. 초보자용 안내에서는 Git, 가상환경, Streamlit 같은 단어를 앞세우지 않는다.

아래쪽에는 "고급/개발자용 안내" 섹션을 별도로 둔다. 이 섹션에는 직접 `venv`를 활성화해서 실행하는 방법, `streamlit run main.py`, `pip install -r requirements.txt`, 데이터 폴더 백업과 이동, Git 관련 주의사항을 모은다. 초보자용 본문과 물리적으로 분리해서 처음 보는 사람이 스크롤 초반에 복잡한 내용을 마주치지 않게 한다.

## Content Rules

- Windows 기준으로 쓴다.
- 첫 실행과 이후 실행을 분리해서 설명한다.
- 실제 저장소 상태와 맞는 파일명만 쓴다.
  - `run_novel_autowriter.bat`
  - `.env.example`
  - `requirements.txt`
- 앱이 지원하는 현재 방식만 안내한다.
  - Gemini CLI OAuth 로그인 후 사용 가능
  - Google API Key 입력 후 사용 가능
- 문장은 짧게 쓰고, 한 단계에 한 행동만 적는다.
- 어려운 단어를 쓰면 바로 쉬운 설명을 붙인다.

## Verification

- 문서 내용이 실제 파일 구조와 맞는지 확인한다.
- 초보자 섹션만 읽어도 설치와 실행 순서를 이해할 수 있는지 확인한다.
- `README.md`가 Markdown으로 깨지지 않는지 확인한다.
