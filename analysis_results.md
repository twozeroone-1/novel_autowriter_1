# 코드베이스 종합 검토 및 잠재적 문제점 분석 리포트

전체적인 코드를 검토한 결과, 현재의 로직과 아키텍처는 프로토타입 단계로서는 매우 훌륭하게 작동하지만, 향후 프로젝트가 커지거나 장기 방문(연재) 횟수가 늘어날 때 발생할 수 있는 잠재적 이슈들을 별표(★) 수로 나누어 정리해 드립니다.

## 🚨 심각도 매우 높음 (★★★)

### 1. 탭 5. 무한 자동 연재(Auto Mode)의 브라우저 종속성
- **현재 구조**: Streamlit의 `st.session_state`와 `time.sleep()`, `st.rerun()`에 의존하여 백그라운드 태스크를 흉내 내고 있습니다.
- **문제점**: 이 앱을 브라우저 탭에 계속 띄워두지 않으면(즉, 창을 닫거나 컴퓨터가 절전모드로 들어가면) **자동 연재가 즉각 중단**됩니다. Streamlit은 사용자의 브라우저 세션이 끊기면 스크립트 실행을 중단합니다.
- **해결 방안**: 향후 진정한 "무한 자동 연재"를 원하신다면, Streamlit은 단순히 설정을 보는 화면으로만 두고, 셀러리(Celery), 혹은 별도의 백그라운드 파이썬 스크립트(`while True:`)를 돌려 파일을 생성하게 만드는 아키텍처 분리가 필요합니다.

## ⚠️ 심각도 중간 (★★☆)

### 2. 등장인물 JSON 파싱 안정성 (`core/generator.py` 및 `core/context.py`)
- **현재 구조**: AI가 캐릭터 목록을 JSON 텍스트 배열로 반환하면, 이를 `json.loads`로 파싱하여 리스트에 넣고, 후속 과정에서 Pydantic의 `Character(**c)`로 변환합니다.
- **문제점**: 만일 AI가 `id`, `name`, `role`, `description`, `traits` 중 하나라도 누락하거나, 콤마(,)를 실수로 빠뜨리는 등 JSON 형식을 완벽히 지키지 않으면 프로그램이 즉시 에러(`ValidationError` 또는 `JSONDecodeError`)를 뿜으며 멈춥니다.
- **해결 방안**: 
  - Gemini API 호출 시 **Structured Outputs (JSON Schema 기반 생성)** 기능을 강제하여 무조건 포맷에 맞게만 출력되도록 묶거나, 
  - `ContextManager.get_character_context` 부분에 `try-except` 블록을 두어 변환 실패 시 무시하거나 기본값을 사용하게 우회해야 합니다.

### 3. Streamlit UI 상태(State) 관리의 복잡성과 리렌더링 버그 가능성 (`main.py`)
- **현재 구조**: 프로젝트를 바꾸거나, AI가 문서를 생성할 때마다 기존 텍스트 입력창과 충돌을 막기 위해 `del st.session_state['key']`를 사용하여 변수를 강제로 지운 뒤 화면을 갱신(`st.rerun()`)하고 있습니다.
- **문제점**: 이는 앱이 복잡해질수록 상태 관리 꼬임(State Tangle) 현상을 유발합니다. 화면이 번쩍이거나, 사용자가 입력 중인 텍스트가 삭제되는 의도치 않은 버그가 재발할 위험이 높습니다.
- **해결 방안**: 강제로 변수를 `del` 하기보다는, 콜백 함수(`on_change`)나 Streamlit Form을 활용하는 형태로 리팩토링하는 것을 권장합니다.

## 💡 심각도 낮음 / 장기적 개선점 (★☆☆)

### 4. API 키 보안 및 배포 환경 (`main.py`)
- **현재 구조**: `.env` 파일을 직접 읽고 쓰며 API 키를 관리합니다.
- **문제점**: 로컬 PC 환경에서는 문제가 되지 않지만, 향후 Streamlit Community Cloud 같은 클라우드 환경에 앱을 배포하게 될 경우, 서버가 재시작되면 파일이 날아가 키 설정이 계속 초기화되는 문제가 발생할 수 있습니다. 
- **해결 방안**: 배포 시에는 Streamlit의 `st.secrets` 기능을 사용해야 합니다.

### 5. 토큰 제한과 요약본 길이 (`core/context.py`)
- **현재 구조**: 과거 줄거리(`summary_of_previous`)가 3000자를 넘어가면 AI에게 다시 초압축을 지시하여 길이를 관리합니다.
- **문제점**: 3000자 자체는 Gemini Flash 모델(최대 100만~200만 토큰 스펙)에게 매우 적은 양입니다. 혹시나 압축 로직 자체에서 LLM이 텍스트 손실을 일으킬 위험이 있으므로, 임계값을 3,000자보다 훨씬 넉넉하게(ex: 15,000자) 늘려두셔도 요즘 LLM 컨텍스트 크기에서는 전혀 무리가 없습니다.

---

**[결론]**
현 상태로 개인이 로컬 PC에서 창을 띄워두고 사용하는 용도로는 **큰 문제 없이 아주 잘 작동**합니다. 당장 급하게 수정해야 하는 치명적인 버그(Error)는 발견되지 않았습니다. 
다만 나중에 "창을 닫아두고 싶다"거나 "UI가 가끔 초기화되는 기분이 든다"고 느끼실 때 **1번**과 **3번**을 가장 먼저 개선 타겟으로 삼으시면 완벽하실 것입니다!
