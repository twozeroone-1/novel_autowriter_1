# Novel Autowriter

AI를 활용하여 자동으로 소설을 작성해주는 프로그램입니다.

## 🚀 설치 및 실행 방법

### 1단계: 코드 가져오기 (Clone)

터미널(또는 명령 프롬프트)을 열고 아래 명령어를 입력하여 코드를 다운로드합니다.

```bash
git clone https://github.com/twozeroone-1/novel_autowriter_1.git
cd novel_autowriter_1
```

### 2단계: 가상환경 설정 및 라이브러리 설치

Python 가상환경(venv)을 만들고, 프로그램 실행에 필요한 패키지들을 한 번에 설치합니다.

```bash
# 가상환경 생성 (Windows)
python -m venv venv

# 가상환경 활성화 (Windows)
venv\Scripts\activate

# 필수 패키지 설치
pip install -r requirements.txt
```

### 3단계: 환경설정 (LLM 실행 방식 설정)

이 프로그램은 두 가지 방식 중 하나로 LLM을 실행할 수 있습니다.
- `google_api` (기본): Google API Key 사용
- `gemini_cli_oauth`: Gemini CLI OAuth 세션 사용 (API 키 불필요)

1. 다운로드 받은 폴더 안에 있는 `.env.example` 파일을 복사하여 `.env` 라는 이름으로 파일을 만듭니다.
2. 새로 만든 `.env` 파일을 메모장 등으로 열어서 `LLM_PROVIDER`를 선택합니다.
3. `google_api` 모드라면 `GOOGLE_API_KEY`에 실제 키를 넣습니다.
4. 💡 **팁 (다중 키 지원):** 트래픽 초과를 대비해 키를 여러 개 넣고 싶다면, 쉼표(`,`)로 구분하여 나열해 주세요. 앞의 키가 막히면 다음 키로 자동 우회(Fallback)합니다.

```env
# .env 파일 수정한 예시
LLM_PROVIDER="google_api"
LLM_FALLBACK_TO_API="true"
GOOGLE_API_KEY="AIzaSy...진짜키1, AIzaSy...진짜키2, AIzaSy...진짜키3"
GEMINI_MODEL="gemini-3-flash-preview"
```

Gemini CLI OAuth 모드를 쓰려면:
```bash
gemini
# 브라우저 로그인 완료 후 앱에서 LLM 실행 방식 = Gemini CLI OAuth 선택
```

`gemini_cli_oauth` 모드에서 실패 시 `LLM_FALLBACK_TO_API="true"`이면 Google API Key 모드로 자동 재시도합니다.

### 4단계: 프로그램 실행

모든 준비가 끝났습니다! 아래 명령어로 프로그램을 실행하세요.

```bash
streamlit run main.py
```

처음 설치가 끝난 후, 나중에 컴퓨터를 껐다 켜고 다시 소설 작업을 이어서 하고 싶을 때는 딱 **두 줄**의 명령어만 순서대로 입력하면 됩니다.

1. VS Code나 터미널에서 `novel_autowriter` (본인의 작업 폴더)를 열어줍니다.
2. 터미널(명령 프롬프트)창에 아래 두 줄을 순서대로 입력하고 실행합니다.

```bash
# 1. 가상환경 켜기 (필수!)
venv\Scripts\activate

# 2. 프로그램 실행하기
streamlit run main.py
```

## 💾 (필독) 다른 컴퓨터로 작업 내역(소설 내용) 옮기기

GitHub(깃허브)에는 프로그램의 '뼈대 코드'만 올라가며, 작가님이 실제로 작성한 세계관, 설정, 마크다운 원고 초안 등은 보안상의 이유로 업로드되지 않습니다. (**`data/` 폴더는 제외됨**)

따라서 **집(A컴퓨터)에서 쓰던 소설을 노트북(B컴퓨터)에서 이어서 쓰고 싶다면**, 아래 절차를 반드시 거쳐야 합니다.

1. A컴퓨터의 프로그램 폴더 안에 있는 **`data` 폴더 전체를 복사**합니다. (USB, 구글 드라이브, 카카오톡 내게쓰기 등 활용)
2. B컴퓨터에 위 "설치 및 실행 방법" 1~3단계를 똑같이 완료합니다.
3. B컴퓨터의 프로그램 폴더 안쪽에, 1번에서 복사해 온 **`data` 폴더를 그대로 붙여넣기(덮어쓰기)** 합니다.
4. 프로그램을 실행하면 A컴퓨터에서 작업하던 폴더 내용과 데이터가 완벽하게 똑같이 뜹니다!

## 📖 사용 설명서 (작업 방식 안내)

이 프로그램은 작가님의 작업 스타일에 맞춰 **3가지 연재 방식(수동 / 반자동 / 완전 자동)**을 지원합니다. 
모든 작업의 시작점은 **[1] 프로젝트 통합 설정**에서 세계관과 설정값을 꼼꼼히 채워넣는 것입니다.

### ✍️ 방식 A: 수동 집필 모드 (세밀하고 깐깐한 제어)
AI를 영리한 보조 작가로 활용하며, 작가님이 직접 내용을 하나하나 검수하고 다듬는 방식입니다.
* **추천 흐름:** `[1] 프로젝트 설정` ➡️ `[2] 회차 생성` ➡️ `[3] 원고 검수`
1. **[1] 탭**에서 `STORY BIBLE`, `STYLE GUIDE` 등 작품의 기초 뼈대를 작성하고 저장합니다. (등장인물 추출도 여기서 진행합니다.)
2. **[2] 탭**으로 이동하여 이번 회차에 일어날 사건(지시사항)을 입력하고 **초안**을 생성하여 저장합니다.
3. **[3] 탭**으로 이동하여 방금 생성된 초안의 **검수**를 요청합니다. AI 편집자의 뼈아픈 피드백을 확인하고, 이를 자동으로 반영한 **수정본**을 저장하면 한 회차가 완성됩니다!

### 🦾 방식 B: 반자동 연재 모드 (원클릭 파이프라인)
매번 생성 버튼을 누르기 귀찮은 분들을 위해, [초안 생성 ➡️ 편집자 검수 ➡️ 퀄리티업 수정본 작성 ➡️ 다음 회차 요약 갱신] 까지의 전체 과정을 버튼 한 번으로 처리합니다.
* **추천 흐름:** `[1] 기본 세팅` ➡️ `[4] 반자동 연재 모드`
1. 처음 한 번만 **[1] 탭**에서 기본 세계관을 세팅합니다.
2. **[4] 탭**으로 이동하여 제목과 전개 지시사항만 입력한 뒤 **🚀 파이프라인 가동** 버튼을 누르고 커피 한 잔을 마시고 옵니다.
3. 결과물(수정본, 리포트)이 도출되면 화면을 읽어보고, **다음 회차를 위한 설정 갱신(상태창)**을 가볍게 다듬어준 뒤 다음 회차로 넘어갑니다.

### 🤖 방식 C: 완전 자동화 연재 모드 (무한루프 공장)
설정된 주기(예: 60분)마다 AI가 이전 회차의 요약을 바탕으로 스스로 전개를 추론하여, 끝없이 다음 회차를 연재하는 궁극의 방목형 모드입니다.
* **추천 흐름:** `[1] 완벽한 세팅` ➡️ `[5] 자동화 연재 모드`
1. **[1] 탭**에서 AI가 절대 엇나가지 않도록 `STORY BIBLE`과 `CONTINUITY(고정 설정)`를 최대한 탄탄하게 세팅합니다.
2. **[5] 탭**으로 이동하여 원하는 **연재 주기(예: 60분)** 와 목표 글자 수를 입력합니다.
3. **🚀 무한 자동 연재 시작** 버튼을 누릅니다.
4. 이제 프로그램은 켜두기만 하면, 지정한 시간이 지날 때마다 이전 상황을 이어받아 끝없이 소설을 찍어냅니다! (중지 버튼을 누를 때까지 영원히 반복됩니다.)

## 🆕 신규 탭 안내

### [6] 아이디어/제목 추천
- 플랫폼 + 관심 키워드를 입력하면 아이디어 5개, 제목 후보, 펀치라인을 생성합니다.

### [7] 대형 플롯 설계
- 300화 기준, 30화 단위 대사건 10개를 생성합니다.
- 생성 결과는 버튼으로 `config.json`에 저장 가능하며, 회차 생성 탭에서 선택 반영할 수 있습니다.

### [2] 회차 생성 탭의 플롯 반영 옵션
- 체크박스로 플롯 반영 여부를 선택할 수 있습니다.
- 강도 옵션:
  - `loose`: 느슨히 참고
  - `balanced`: 권장값
  - `strict`: 플롯 우선

## 🔐 앱 접근 토큰(선택)

공개 배포 시 최소 보호가 필요하면 `APP_ACCESS_TOKEN`을 설정하세요.

```env
APP_ACCESS_TOKEN="원하는_긴_토큰값"
```

- 토큰이 설정되어 있으면 앱 시작 시 인증 화면이 먼저 나옵니다.
- 로컬 `.env` 또는 Streamlit `st.secrets`로 설정할 수 있습니다.

## 🛠️ 문제 발생 시 대처 가이드

### 0) 프로젝트 생성 시 이름 규칙
- 프로젝트 이름은 `한글/영문/숫자/공백/_/-`만 허용됩니다.
- `/`, `\`, `..` 같은 경로 문자는 보안상 차단됩니다.

### 1) LLM 요청이 오래 걸리거나 멈춘 것 같을 때
1. 탭 `[5] 자동화 연재 모드`로 이동
2. `🧯 강제 중지` 버튼 클릭
3. 필요하면 `🔓 락 해제` 클릭
4. 다시 시작 전 API 키/모델/지시사항을 점검

### 2) 자동 연재가 꼬였을 때
1. `♻️ 체크포인트 복구` 버튼 클릭
2. 복구는 락 재획득에 성공해야 재개됩니다.
3. 복구가 안 되면 `🧯 강제 중지` 후 재시작

### 3) API 오류(429/503/timeout)가 자주 뜰 때
1. 키를 쉼표로 여러 개 등록 (Fallback)
2. 모델을 더 가벼운 모델로 변경
3. 생성 글자 수를 낮춰서 재시도

### 5) Gemini CLI OAuth 모드에서 실패할 때
1. 터미널에서 `gemini`를 실행해 로그인 상태를 갱신
2. `~/.gemini/oauth_creds.json` 존재 여부 확인
3. `gemini --help`가 동작하는지 확인 (CLI 설치 확인)

### 4) 모바일에서 화면이 답답할 때
1. 긴 텍스트 영역은 접고 필요한 탭만 사용
2. 자동 연재보다는 수동/반자동 모드를 우선 사용
