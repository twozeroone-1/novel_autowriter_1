# Plot Flow Expansion Design

## Goal

저장된 대형 플롯을 선택적으로 `원고 검수`, `[4] 반자동 연재 모드`, `[5] 자동화 연재 모드`에도 연결한다.

## Scope

- `원고 검수`에서 플롯 반영 체크박스와 강도 선택을 추가한다.
- `반자동 연재 모드`에서 플롯 반영 체크박스와 강도 선택을 추가한다.
- `자동화 연재 모드` 설정 화면에 플롯 반영 체크박스와 강도 선택을 추가한다.
- 저장된 플롯이 없으면 각 화면에서 옵션을 비활성화하고 안내 문구를 보여준다.

## Design

### 1. 검수 경로

`Reviewer.review_chapter()`와 `Reviewer.revise_draft()`가 선택적으로 플롯 블록을 프롬프트에 포함한다. 생성 경로와 같은 형식의 플롯 블록을 재사용해, 검수 기준과 수정 방향이 장기 플롯과 어긋나지 않게 한다.

UI에서는 `[3] 원고 검수`에 체크박스와 강도 선택을 추가한다. 검수 보고서를 생성할 때 사용한 옵션은 수정본 생성에도 그대로 재사용한다.

### 2. 반자동 경로

`Automator.run_single_cycle()`가 `include_plot`, `plot_strength`를 받아 생성 단계에 전달한다. `[4] 반자동 연재 모드`는 현재의 회차 생성 탭과 유사한 플롯 토글 UI를 제공한다.

### 3. 자동화 경로

자동화는 작업별이 아니라 프로젝트 단위 기본 설정으로 처리한다. `automation/config.json`에 `generation_options.include_plot`, `generation_options.plot_strength`를 저장하고, 런타임이 이를 읽어 `run_single_cycle()` 호출에 전달한다.

UI에서는 `[5] 자동화 연재 모드` 설정 영역에 체크박스와 강도 선택을 추가한다.

## Data Flow

1. 사용자가 `[1] 프로젝트 통합 설정 > 대형 플롯`에서 플롯을 저장한다.
2. 각 실행 화면이 `ContextManager.get_plot_outline()`으로 저장된 플롯 존재 여부를 확인한다.
3. 사용자가 플롯 반영을 켜면 생성/검수 프롬프트에 플롯 블록을 포함한다.
4. 자동화 모드는 저장된 설정을 런타임이 읽어 동일한 방식으로 적용한다.

## Error Handling

- 저장된 플롯이 없으면 체크박스는 비활성화한다.
- 플롯 옵션이 꺼져 있으면 기존 동작을 그대로 유지한다.
- 자동화 설정 파일에 새 필드가 일부만 있어도 기본값을 안전하게 보완한다.

## Testing

- Reviewer 프롬프트가 플롯을 포함/미포함하는지 테스트한다.
- Automator가 플롯 옵션을 생성 단계에 전달하는지 테스트한다.
- AutomationRuntime이 저장 설정을 `run_single_cycle()`에 전달하는지 테스트한다.
- AutomationStore가 새 중첩 기본값을 안정적으로 로드하는지 테스트한다.
